# HSL Mockup Modules

This project is a static HSL ticket screen/PWA. The page is intentionally split into visual modules so future changes can be made in a targeted way.

## Entry Files

- `index.html`: Main deployable page for GitHub Pages/PWA.
- `styles.css`: Main styling source.
- `hsl-ticket-single.html`: Self-contained test file with CSS and JS embedded.
- `manifest.webmanifest`: App name, icon, display mode, and install metadata.
- `service-worker.js`: PWA cache. Bump `CACHE_NAME` whenever visible files change.

When changing `index.html` or `styles.css`, regenerate `hsl-ticket-single.html` so the one-file version stays current.

## Visual Modules

### Top Tabs

- HTML: `.ticket-tabs`, `.tab-button`
- CSS: `.ticket-tabs`, `.tab-button`, `.tab-button.active::after`
- Current behavior: `Your tickets` is active; blue underline marks the selected tab.

### Ticket Card

- HTML: `.ticket-card`
- CSS: `.ticket-card`
- Contains QR code, ticket number, subscription badge, ticket title, details, and zone card.

### Side Punch Holes

- HTML: `.side-notch`, `.notch-left`, `.notch-right`
- CSS: `.side-notch`, `.notch-left`, `.notch-right`
- Design intent: natural half-circle punch holes with the same color as the surrounding page background.
- Avoid adding borders here; borders make the diameter/edge visible.

### QR Code

- HTML: `.qr-code`
- JS: QR-like grid generation near the bottom of `index.html`
- CSS: `.qr-code`, `.qr-code span`, `.qr-code .is-dark`
- Current grid: `45 x 45`, `145px` square.
- Finder corners are generated by `addFinder`.

### Subscription Badge

- HTML: `.subscription-badge`
- CSS: `.subscription-badge`, `.subscription-badge svg`, `.gear-hole`
- Design intent: blue gear icon plus bold `SUBSCRIPTION` text inside a light bordered pill.

### Ticket Title Row

- HTML: `.ticket-title`
- CSS: `.ticket-title`, `.ticket-title strong`, `.title-rest`
- Design intent: only `SEASON TICKET` is bold; `- Auto-renewing s...` is regular weight.

### Validity Details

- HTML: `.details-row`, `.label`, `.value`, `.status-value`, `.customer`
- CSS: matching class names.
- Design intent: gray labels, strong values, green status dot.

### Zone Card

- HTML: `.zone-card`, `.zone-pattern`, `.zone-color-panel`, `.zone-copy`, `.zone-letters`, `.zone-label`
- CSS: `.zone-card`, `.zone-pattern`, `.zone-pattern-spin`, `.zone-copy`
- Left and right sections are a single shared background color; right side has a semi-transparent gray overlay.
- Current right overlay: `rgba(249, 249, 251, 0.985)`.
- Current pattern speed: `7s` per rotation.
- Current pattern stroke: pure white, `17px`.
- Current pattern ellipse size: `rx=7.5`, `ry=12`.
- Design target: each ellipse's inner tip should meet the matching octagon vertex smoothly. If `ry` changes, move each ellipse center along the center-to-vertex direction by the same `ry` distance.

### Zone Color Controls

- HTML: `.color-controls`, `.color-button`
- JS: button click handler near the bottom of `index.html`
- CSS variable: `--zone-color`
- Storage key: `hsl-zone-color` in `localStorage`, used to restore the last selected color on reopen.
- Current colors: blue, red, orange (`#f36a25`, brighter/saturated sample from the target ticket image), green, yellow.
- Orange color migration: old saved `#eb6e34` values are mapped to `#f36a25` on reopen.

### Bottom Navigation

- HTML: `.bottom-nav`
- CSS: `.bottom-nav`, `.bottom-nav button`, `.bottom-nav svg`
- Design intent: fixed bottom navigation, `TICKETS` selected.

## Font Audit Notes

Primary font stack:

```css
-apple-system, BlinkMacSystemFont, "Segoe UI", Arial, Helvetica, sans-serif
```

Important weights:

- Tabs: bold, `800`.
- Ticket number prefix: `900`; remaining digits regular.
- Subscription text: `900`.
- `SEASON TICKET`: `900`.
- `- Auto-renewing s...`: regular, `400`.
- Validity values and `Adult`: `800`.
- `AB`: `900`.
- Bottom nav labels: `900`.

The reference screenshot appears to use an iOS-style rounded sans. Browser rendering may not be pixel-identical to the native HSL app, but this stack is the closest system-font approximation for iPhone/Safari.

## PWA Notes

GitHub Pages serves the app publicly. After changes are pushed:

1. GitHub Pages redeploys the static files.
2. The phone-installed PWA may still use cached files briefly.
3. Bumping `CACHE_NAME` in `service-worker.js` forces the app shell cache to refresh.
4. On iPhone, close and reopen the home-screen app if the update does not appear immediately.
